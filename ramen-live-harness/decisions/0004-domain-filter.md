# ADR-0004: 라면/라멘 도메인 식별 — 코드 1차 필터 + 키워드 + 보정 리스트

- **상태**: 채택
- **일시**: 2026-06-18
- **관련**: phase M1·M4, `docs/TECHNICAL` §1.1·§3.1

## 맥락
Q3(도메인 필터 매핑) 확인 결과, 제품·음식점 어느 쪽에도 "라면/라멘" 전용 코드가 없다.

- **제품(식품공전)**: 면류 식품유형은 생면·숙면·건면·유탕면. 봉지/컵 라면은 대개 **유탕면**, 비유탕 라면은 **건면**. "라면"은 식품유형이 아니라 제품명 개념. 코드 출처: 식품안전나라/행정표준코드(code.go.kr) 식품안전품목분류, 데이터의 품목유형(식품유형) 필드.
- **음식점(일반음식점 업태구분명)**: 한식·중식(중국식)·일식·분식·경양식·횟집·냉면집·김밥(도시락) 등. 라멘집은 대개 **일식**, 한국식 라면/분식은 **분식**. "라면/라멘" 업태는 없음. 일반음식점 + 휴게음식점 모두 대상(컵라면 제공 편의점·휴게소는 제외 규정 존재).

## 결정
2단계 + 보정으로 식별한다.

- **제품**: ① 식품유형 = 면류 계열(유탕면·건면 중심)로 범위 축소 → ② 제품명 키워드("라면","라멘","면","사발","컵") → ③ 보정 리스트(오탐 제외/누락 추가).
- **음식점**: ① 업태 = 일식·분식(+기타·한식 일부) 후보 → ② 상호(사업장명) 키워드("라멘","라면","麺","멘야") → ③ 보정 리스트.
- 보정 리스트는 버전 관리하며, 오탐/누락 케이스를 픽스처로 축적한다(M1·M4 DoD의 오탐·누락 리포트 게이트와 연동).

## 근거
전용 코드가 없어 코드만으로는 분류 불가, 키워드만으로는 오탐(예: "냉면", "라면땅" 과자)이 과다. 코드로 범위를 좁히고 키워드로 선별하되 불완전성을 보정 리스트로 흡수하는 것이 유지가능한 절충이다.

## 결과 / 트레이드오프
- 이득: 합법 데이터(식약처·LOCALDATA)만으로 도메인 구성. 정밀도를 보정 리스트로 점진 개선.
- 비용: 보정 리스트 유지 비용. 신제품·신규 업소는 갱신 시 누락 가능 → 오탐·누락 리포트로 모니터링.

## 검토한 대안
- 코드만 필터 — 라면 외 면류·일식 전체 혼입. 기각.
- 키워드만 필터 — 오탐 과다. 기각.
