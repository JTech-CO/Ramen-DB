# ADR-0001: 데이터를 3-tier로 분할하고 Tier 2를 비영속으로 둔다

- **상태**: 채택
- **일시**: 2026-06-18
- **관련**: phase M5, INV-4, `docs/TECHNICAL` §0·§1.2

## 맥락
수집 대상 데이터는 저장·재배포 가능성에서 균질하지 않다. 상용 검색/쇼핑/지도 API는 결과의 영구 저장과 재배포를 약관으로 제한하며(조회 시점 표시 + 출처표기 모델), 외부 플레이스·블로그 크롤링은 ToS·저작권 위반 소지가 있다.

## 결정
데이터를 3-tier로 분할한다. Tier1(공공데이터)만 영속 적재·재배포한다. Tier2(가격·구매링크·외부 리뷰)는 비영속으로 두고 런타임 조회 + 약관 허용 캐시(TTL) + 출처·조회시각 표기만 한다. Tier3(레시피·맛집 큐레이션)는 사이트 자체 UGC·운영자 검수로만 채운다.

## 근거
약관 변동·법적 리스크를 아키텍처로 차단한다. 핵심 자산은 합법적으로 재배포 가능한 공공데이터에 집중시켜 제품의 방어선을 명확히 한다.

## 결과 / 트레이드오프
- 이득: 약관 변경에 구조적으로 강건. 핵심 DB는 합법·재배포 가능.
- 비용: 가격은 항상 실시간 조회(지연·호출비). Tier2 비영속을 INV-4 + 패키지 경계(`PriceQuote`는 `apps/web`에만)로 강제해야 한다.

## 검토한 대안
- 전 데이터 적재(가격 포함 주간 스냅샷) — 약관 위반·법적 리스크. 기각.
