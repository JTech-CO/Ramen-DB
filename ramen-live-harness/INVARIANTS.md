# INVARIANTS.md — 불변식 레지스트리

> 불변식은 "절대 일어나면 안 되는 일"이다. 일반 DoD와 달리, **1건이라도 위반되면 통과 금지**. DoD와 레드라인은 여기 ID(INV-n)를 참조한다. 새 불변식은 ID를 부여해 추가하고, 폐기 시 줄을 지우지 말고 상태를 `폐기`로 표시한다.

## 등록 규칙
- 검증 가능한 형태로 쓴다(명령·테스트·자동 점검으로 확인되게).
- "범위"에 어느 phase/모듈에 적용되는지 명시.
- DoD에서 `(INV-4 준수)`처럼 인용한다.

## 레지스트리
| ID | 불변식 | 범위 | 검증 방법 | 상태 |
|---|---|---|---|---|
| INV-1 | 시크릿(`.env`·서비스키·토큰)을 커밋하지 않는다 | 전역 | pre-commit/CI 시크릿 스캔, 리뷰 | 활성 |
| INV-2 | 대용량 산출물(스냅샷 바이너리·빌드물)을 커밋하지 않는다 | 전역 | `.gitignore`·CI 크기 점검 | 활성 |
| INV-3 | 모듈 경계를 위반하는 상호 import가 없다 | 전역 | eslint 경계 룰(`docs/FILE_TREE`) | 활성 |
| INV-4 | Tier 2 데이터(가격·구매링크·외부 리뷰)를 영속 저장소·스냅샷에 기록하지 않는다. 런타임 조회+약관 허용 캐시(TTL)만 허용하며 응답에 출처·조회시각을 표기한다 | Tier2 경로(`apps/web`) | 영속 스키마에 `PriceQuote` 부재 + 경계 룰 import 차단 + 저장 호출 테스트 | 활성 |
| INV-5 | `품목제조보고번호`는 제품 불변 PK다. 제품명·제조사명 변경에도 동일 레코드를 유지한다 | 제품 데이터(`core-domain`) | 이름 변경 픽스처로 스냅샷 간 PK 안정성 테스트 | 활성 |
| INV-6 | 모든 `status`는 `source`와 `confidence`를 동반한다. confidence<high를 단정형으로 표기하지 않는다 | 제품 status(`core-domain`,`web`) | 스키마 필수필드 검증 + UI 표기 규칙 테스트 | 활성 |
| INV-7 | Tier 1 전체 데이터셋은 idempotent full-snapshot로 재생성 가능하다. 동일 입력→동일 산출(정렬·해시 일치) | 스냅샷 파이프라인(`pipeline`) | 동일 입력 2회 실행 산출 해시 동일 | 활성 |
| INV-8 | UGC(레시피·꿀조합·맛집 큐레이션)는 사이트 자체 제출분만 수용한다. 외부 블로그·플레이스를 크롤링하지 않는다 | 콘텐츠 인입(`web`) | 인입 경로 리뷰 + 크롤러 부재 점검 | 활성 |
| INV-9 | 외부 출처 데이터는 표시 시 출처를 명시한다(공공데이터 라이선스·상용 API 약관 준수) | 전 표시 경로 | 렌더 출처표기 테스트 | 활성 |
| INV-10 | 쿠팡 파트너스 등 제휴 링크가 노출되는 모든 화면에 공정위 기준 대가성 문구를 명확·확정형으로 눈에 띄게 표기한다(조건부·불확정 표현 금지, 제목 또는 첫머리) | 제휴 표시 경로(`apps/web`) | 링크 노출 화면의 고지 문구 존재·위치 테스트 | 활성 |
