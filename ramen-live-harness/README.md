# Harness Expand Pack — ramen.live

확장·고도화된 다중 파일 작업 하네스. 단일 파일 하네스가 "운영 규율"만 담았다면, 이 팩은 **에이전트 계약·상태 인계·불변식·게이트·phase별 상세·결정 기록·런북·백서**를 역할별 파일로 분리해 종합한 것이다. 복잡하고 장기적인 프로젝트를 Claude Code로 진행할 때, 세션이 끊겨도 일관성과 검증 규율이 유지되도록 한다.

> 표기: `{{...}}` 는 프로젝트에 맞게 채운다. *(가이드: ...)* 줄은 작성 지침이므로 채운 뒤 삭제한다.

## 단일 파일 하네스와의 차이
- **관심사 분리**: 매 세션 바뀌는 것(`PROGRESS.md`)과 거의 안 바뀌는 것(`HARNESS.md`·`INVARIANTS.md`)을 분리.
- **정밀 참조**: 에이전트를 정확히 한 파일로 보낼 수 있다(예: "M3 작업 시 `phases/M3.md`만 본다").
- **성장하는 파일**: 런북·결정 로그는 시간이 지나며 누적되므로 별도 파일.
- **증거 기반 완료**: 게이트 통과를 주장할 때 명령과 출력을 phase 파일/`PROGRESS.md`에 남긴다.

## 세션 프로토콜 (요약 — 상세는 `HARNESS.md` §0)
1. 시작: `CLAUDE.md`(자동) → 이 README → `PROGRESS.md` → 현재 `phases/Mn.md` → 관련 `docs/` 절.
2. 작업: 한 번에 한 phase. 작업 단위마다 그 phase의 검증 명령으로 게이트 확인.
3. 종료: `PROGRESS.md` 갱신(완료·증거·다음 할 일·미결·결정 링크) → 커밋.

## 도입 방법
1. 이 팩을 레포 루트(또는 `harness/`)에 복사.
2. `{{...}}` 채우고 *(가이드: ...)* 줄은 삭제.
3. `gates/DOD_GUIDE.md` §3 아키타입에서 필요한 phase만 골라 `phases/Mn.md`로 인스턴스화.
4. `INVARIANTS.md`에 프로젝트의 절대 금지선을 ID로 등록.
5. 첫 세션 시작.
