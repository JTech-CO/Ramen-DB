# CLAUDE.md — ramen.live 에이전트 계약

> Claude Code가 세션마다 자동으로 읽는 최우선 문서. 여기 규칙이 다른 모든 지시에 우선한다. 상세 절차는 `HARNESS.md`, 불변식 본문은 `INVARIANTS.md`.

## 프로젝트 한 줄
국내 라면(봉지/컵 가공라면 + 전국 라면·라멘 음식점)의 제품·영양·레시피·판매상태·구매경로를 통합 제공하는 데이터 플랫폼. 데이터는 3-tier(Tier1 공공데이터=창고형 / Tier2 상용 API=실시간 비영속 / Tier3 UGC)로 분할 관리한다.

## 매 세션 행동
1. 먼저 `PROGRESS.md`를 읽어 현재 phase·다음 할 일·미결을 파악한다.
2. **한 번에 한 phase.** 해당 `phases/Mn.md`의 진입조건(DoR)을 만족할 때만 시작한다.
3. **완료 = 측정 가능한 게이트(DoD) 전부 통과.** "에러 없이 돈다"는 완료가 아니다.
4. 게이트 통과를 주장할 때는 **명령과 출력을 증거로** phase 파일/`PROGRESS.md`에 남긴다.
5. 종료 시 `PROGRESS.md`를 갱신하고 커밋한다.

## 레드라인 (절대 금지 — 위반 시 즉시 멈춤)
- **Tier 2 데이터(가격·구매링크·외부 리뷰)를 영속 저장**하거나 스냅샷에 적재(INV-4).
- **status를 source·confidence 없이 기록**하거나 confidence<high를 단정형('단종됨')으로 표기(INV-6).
- 외부 블로그·플레이스 **크롤링**으로 콘텐츠 수집(INV-8).
- 품목제조보고번호 외의 키로 제품 PK를 잡거나 PK를 가변값으로 둠(INV-5).
- 테스트 삭제·약화 또는 게이트 수치 임의 하향으로 "통과" 위장.
- 불변식(`INVARIANTS.md`)이 깨진 채 다음 phase 진행.
- 아키텍처·패키지 경계 임의 변경(사용자 승인 없이). 경계는 `docs/FILE_TREE`.
- 시크릿(`.env`·서비스키)·대용량 산출물 커밋.
- 가드레일 약화(약관 위반 소지·과한 권한).

## 멈춤(STOP) — 다음이면 작업을 멈추고 보고한다
- 같은 실패를 서로 다른 방법으로 3회 시도해도 미해결.
- DoD 게이트를 못 넘는데 우회 충동.
- 불변식을 깨야만 통과 가능(특히 INV-4 비영속, INV-6 status).
- 큰 아키텍처 변경 필요 / 외부 제약(공공데이터 접근·상용 API 약관).
→ 멈출 때 절차는 `HARNESS.md` §2.
